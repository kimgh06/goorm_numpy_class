# Why Numpy?
왜 넘파이인가
연산을 고수준에서 도와줌

실제로 넘파이는 딥러닝에서 가장 중요한 라이브러리
다양한 프레임워크가 있지만 배움에서는 넘파이가 중요
문제는 기본적으로 선형대수에 특화되어 있음

넘파이를 제대로 못따라오는 사람이 있음

어떻게 쓰냐에 따라 같은 연산이라도 성능이 달라짐
앞으로 알고리즘을 넘파이로 제작할 것임.

텐서의 연산과 관련이 있음

scalar 0차
vector 1차
matrix 2차 텐서라 불림
....
나머지는 일반적으로 텐서로 불림

일반적인 n차 텐서를 다룸

그래서 왜 씀?

전부 텐서의 연산임.
이미지는 2차 텐서죠?
근데 컬러면 3차 텐서에요

근데 이게 백장이면 4차 텐서에요

넘파이는 고수준 텐서를 다루기 위해 사용됨.
연산효율도 좋아지는 게 넘파이임

음성도 텐서로 할 수 있어요

벡터로 바꿀 수 있음

컴퓨터는 0,1밖에 몰라요

이 때문에 넘파이에서는 ndarray 객체를 써요
고차원 텐서를 일반적으로 표현하기 위함.

텐서끼리 연산을 해줘야 함.

[1,2,3] * [2,3,4]
이 연산을 편하게 해줄 수 있음.
벡터의 연산이 모두 가능함.

텐서를 다루는 데에 특화되어 있음.
api 도 여기에 맞춰져있음.

가장 강조하는 게 벡터화임.
for 문이 없애는 게 되게 신기하지만 
메모리 레이아웃이 필요.

넘파이를 사용해서 for 문을 없앨 수 있음.
효율성이 높아진다.

몇 만 배 차이가 날 수 있음.

cpu는 한 번에 연산 하나밖에 못함.
cpu 캐시에 메모리 데이터르 가지고 와서 연산하는 데 넘파이는 캐시의 크기를 늘려줍니다.

안 배울 이유가 없음.
머리속에서 그리기 어려운 연산을 많이 하게 될 거임.

다른 라이브러리와 인터페이스 호환이 잘 되어 있어요
중간 다리를 해주는 역할임.

넘파이는 진짜 중요함
딥러닝을 배우기 위한 것은 모두 다룰 것임.

어떻게 배움?
많은 사람이 기본적인 ndarray 특징과 api 사용법을 정리해 놓은 게 많음.
사람을 관찰을 한 결과 심도 있게 사용을 못함.

고수준 텐서를 다루는 힘이 부족함.

코드 이해하느라 숲보단 나무를 보게 되서

중요한 건 텐서에서 어떤 연산을 하는 지 이해하는 게 중요

텐서를 제대로 다루지 못하면 문제점이 하나 있는 데 버그가 나는 코드를 쓰게 됨.

넘파이는 유연해서 버그가 나기 쉬움.

텐서를 이해하는 게 중요함.

프로그래밍은 이해다.

넘파이 라이브러리는 암기가 필요

앞으로 어떻게 사용해야 겠다라는 생각이 들게 해야 함

데이터 과학에 굉장히 힘이 될 것.

지금 부터 텐서를 다루게 될 것이다.

텐서가 머리에 그려져야 함.
